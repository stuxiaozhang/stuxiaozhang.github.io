【CNN_GA】Automatically Designing CNN Architectures Using Genetic Algorithm for Image Classification

本文最后更新于：2022年6月8日

CNN_GA：Automatically Designing CNN Architectures Using Genetic Algorithm for Image Classification

Genetic CNN 的问题：

  1. 阶段的数量是手动预定义的。从这种编码策略中可以清楚地看到，需要多次手动干预，例如每个阶段和总阶段中构建块的数量。由于这些数字与发现的CNN的深度有关，因此需要预定义这些参数，以获得CNN的适当深度。
  2. 每个阶段的构建块参数都不能改变。这种编码策略只能在发现给定CNN的连接时进行查看，不能改变给定CNN的深度。

Basic Idea

  1. 提出了一种变长编码策略和相应的变长交叉算子。

  2. 引入了跳跃连接（The skip connection）增加CNN的深度，从而提高CNN的表现，避免梯度消失问题。此外，这种设计还可以减少搜索空间，以便在有限的时间内获得最佳性能。此外，与其他性能相似的模型相比，该算法所改进的体系结构具有更少的参数。

  3. 完全自动化，不需要人工干预的CNN架构。

  4. 设计一个异步计算组件来加速计算，一个cache组件来缩短适应度评估时间。

Background

现有的CNN架构设计算法可分为两类。首先是“自动+手动调整”CNN架构设计，这意味着仍然需要基于设计CNN架构的专业知识进行手动调整。这一类别包括

  • 遗传CNN方法（Genetic CNN）
  • 层次表示方法（Hierarchical Evolution）
  • 高效架构搜索方法（EAS）
  • 块设计方法（block-QNN-S）
  • 高级神经架构搜索方法（NSANet）

另一种是“自动”CNN架构设计，使用时不需要用户手动调整。

  • 大规模进化方法（Large-scale Evolution）
  • CGP-CNN
  • 神经架构搜索方法（NAS）
  • 元建模方法（MetaQNN）

根据所采用的技术，CNN架构设计也可以分为基于进化算法的架构设计和基于强化学习的架构设计。

  • 基于进化算法：Genetic CNN, Large-scale Evolution, Hierarchical Evolution and CGP-CNN

  • 基于强化学习：NAS、MetaQNN、EAS、Block-QNN-S 和 NSANet

Genetic CNN

编码CNN的整个结构由多个阶段组成。在每个阶段，对一个小型架构进行编码，然后将多个不同的小型架构堆叠起来，形成整个架构。就像NSANet方法一样，为Genetic CNN提供了一个CNN架构框架，而小架构取代了所提供框架中的卷积层，形成了最终的CNN。在每个阶段中，CNN的多个预定义构建块被排序，并且它们的连接被编码。对于这些构建块的排序，第一个和最后一个构建块是手动指定的，其余的都是具有相同设置的卷积层。为了对这些有序构建块之间的连接进行编码，使用了二进制字符串编码方法。 假设有四个构建块，字符串“111-10-0”表示第一个构建块与所有其他构建块都有连接。第二个与第四个构建块没有连接，第四个构建块与第三个构建块相同。

Genetic CNN 的问题：

  1. 阶段的数量是手动预定义的。从这种编码策略中可以清楚地看到，需要多次手动干预，例如每个阶段和总阶段中构建块的数量。由于这些数字与发现的CNN的深度有关，因此需要预定义这些参数，以获得CNN的适当深度。
  2. 每个阶段的构建块参数都不能改变。这种编码策略只能在发现给定CNN的连接时进行查看，不能改变给定CNN的深度。

CNNGA

算法框架

  1. 在进化过程中，用作者提出的变长编码策略对预定义的构建块进行编码，使用预定义的总体大小随机初始化总体。
  2. 当前生成的计数器初始化为零
  3. 开始循环：在计数器未超过预定义的最大生成数之前：
    1. 在进化过程中，在给定的数据集上评估编码CNN特定架构的每个个体的fitness，使用提出的加速组件。
    2. 根据父代个体的 fitness 选择，然后通过包括交叉和变异算子在内的遗传算子生成新的后代。
    3. 然后，通过环境选择从当前种群中选择存活到下一代的个体种群。具体而言，当前种群由父种群和后代种群组成。
    4. 计数器增加1，并继续进化。
  4. 最终选出最好的 fitness 的个体

1. 种群初始化

作者提出了跳跃层，跳跃层由两个卷积层和一个跳跃连接组成。跳跃连接从第一卷积层的输入连接到第二卷积层的输出。如上所述，卷积层的参数是特征映射数、滤波器大小、步长和卷积运算类型。在所提出的编码策略中，我们分别对滤波器大小、步幅大小和卷积运算使用相同的设置。特别地，滤波器和步幅大小分别设置为3×3和1×1，并且仅使用相同的卷积运算。为此，为跳跃层编码的参数是两个卷积层（分别表示为 F1 和 F2）的特征映射的数目。此外，对于filter大小和步幅大小，该编码策略中使用的池化层设置为2×2。为此，为池化层编码的参数仅为池化类型（表示为 P1）。

算法2显示了种群初始化的详细信息：流程是T个个体用相同的方法初始化，然后存储到 $P_0$ 中。

  1. 开始循环：在未超过预定义的之前：
    1. 在个体初始化过程中，首先随机初始化个体的长度 L，表示相应CNN的深度。
    2. 然后，==创建一个包含L个节点的链表==。
      1. 之后，配置每个节点：
        1. 从 (0, 1) 随机生成一个数字 r。
        2. 如果 r<0.5，节点类型设置为1，表示为跳跃层。在跳跃连接层的情况下，随机生成特征图的编号，然后将其分配给node.F1和node.F2。
        3. 否则，节点类型设置为2，表示为池化层。池化类型随机决定：随机生成一个q，当q概率低于0.5时，P1设为最大池化，否则设为平均池化。
      2. 将链表存储到 $P_0$ 中。

在一个跳跃层中使用两个卷积层，丢弃完全连接层的原因：

  1. 全连接层容易导致过拟合现象。为了减少过拟合现象，通常使用dropout随机移除部分连接。但是，每个dropout都会引入一个参数。只有正确指定参数才能使相应的CNN具有良好的性能。同时，全连接层的数量和每个全连接层中的神经元数量也是两个难以调整的参数。
  2. 如果将全连接层合并到所提出的编码策略中，搜索空间将大大扩大，从而增加找到最佳CNN架构的难度。

2. 种群 $P_t$ 中评估个体的 fitness 的流程

给定包含所有用于评估 fitness 的个体的总体 $P_t$，以及CNN的最佳架构要在其上发现的图像分类数据集

适应度函数算法以相同的方式评估 $P_t$ 的每个个体，最后返回包含已评估 fitness 的个体的 $P_t$。

  1. 如果针对初始化的总体即 $P_0$ 进行可行性评估，则会创建一个==全局变量 Cache==，存储不可见体系结构的个体的fitness。

  2. 对于 $P_t$ 中的每个个体（由个体表示），如果在 Cache 中 找到个体，则直接从 Cache 中获取其适应度。否则，对个体进行适应性评估。

    从缓存中查询个体是基于个体的标识符的。理论上，可以使用任意标识符，只要它们能够区分编码不同体系结构的个体。在所提出的算法中，224哈希码被用作相应的标识符，该哈希码已由大多数编程语言在编码体系结构方面实现。

    此外，个体被异步放置在可用的GPU上，这意味着不需要等待下一个个体的适应性评估，直到当前个体的适应性评估完成，而是立即将下一个个体放置在可用的GPU上。

个体的 fitness 评估

  1. 首先，对单个CNN进行解码，并基于给定的图像分类数据集向CNN（第1行）添加分类器。 解码CNN时，将 RELU 和 BN 添加到卷积层的输出中。此外，当跳跃层的空间编号不同于输入数据的空间编号时，将卷积层添加到输入数据中，卷积层具有单位filter和单位stride，但具有特征图的特殊编号。然后，在训练数据上使用随机梯度下降（SGD）算法对CNN进行训练，并在评估数据上计算分类精度（第5行）。
  2. 当训练阶段结束时，对评估数据的最佳分类精度设置为个体的fitness。
  3. 最后，将个体的标识符和fitness关联并放入缓存。

设计这种异步和缓存组件的原因。综上所述，由于CNN上的训练非常耗时，根据特定的体系结构，训练时间从几个小时到几个月不等，因此在所提出的算法中，CNN的设计目的是为了加速安全性评估。具体来说，异步组件是基于GPU的并行计算平台。由于计算梯度的计算性质，深度学习算法通常放置在GPU上以加速训练。事实上，现有的深度学习库 支持在多个GPU上进行计算。然而，他们的并行计算是基于数据并行和模型并行管道。在数据并行管道中，输入数据分为7个小组，每组放在一个GPU上进行计算。原因是一个GPU有限的内存不能同时有效地处理整个数据。在模型并行管道中，一个模型被划分为几个小模型，每个GPU携带一个小模型。原因是一个GPU上有限的计算能力无法运行整个模型。然而，设计的平行管道显然不属于这两条管道中的任何一条，而是基于它们的更高级别。因此，设计这样一个异步组件是为了充分利用GPU计算资源，特别是对于基于种群的算法。此外，如果问题可以划分为几个独立的子问题，则异步组件广泛用于解决大型问题。通过在不同的计算平台上并行执行这些子问题，从而缩短了整个问题的总处理时间。

缓存组件还用于加速可靠性评估，这基于以下考虑：

  1. 存活到下一代的个体，如果其架构没有改变，则无需再次评估可靠性
  2. 已评估的架构可以通过另一代中的变异和交叉操作重新生成。注意，大规模进化的权重继承不能用于第二个考虑。

通常，缓存系统应该认真对待其大小，并提供详细信息来讨论由重复密钥导致的冲突问题。在所提出的算法中，无需进行任何研究。首先，缓存组件类似于地图数据结构，其中该组件中的每个记录都是一个字符串，它将CNN的标识符和适应度值组合在一起。例如，“identifier1=98.12”等记录表示标识符为“identifier1”，其适应值为“98.12”。其次，正如我们所强调的，标识符由224个哈希代码计算，该哈希代码可以生成2224个不同的标识符。在实践中，包括所提出的算法在内的CNN体系结构算法仅评估数千个CNN。显然，我们不需要考虑冲突问题，因为冲突问题不会发生。第三，用于该算法的224哈希码实现将生成长度为32的标识符，而thefitness值是由长度为4的字符串表示的分类精度。缓存组件中的每条记录都是一个长度为37的字符串，使用UTF-8文件编码占据37个字节。显然，即使有数千条记录，缓存文件也只会占用很少的磁盘空间。因此，我们不需要关心缓存组件的大小。

生成子代

生成子代主要的遗传操作是交叉和突变：

  1. 随机生成一个 $r$，根据 $r$ 的大小选择操作

  2. 交叉操作：将生成全部 $|P_t|$ 子代。首先选择两个父母，然后选择两个父母中fitness值更高的那个。这种选择被称为 Binary Tournament Selection(二进制锦标赛选择算法)。然后生成一个随机数 $p_c$ 确定是否进行交叉。如果执行交叉操作，将每个父代个体随机分成两部分，并将父代个体的两部分交换以形成两个后代。否则，将这两个父代个体直接作为后代。

  3. 变异操作：生成一个随机数 $p_m$ 确定是否进行变异。突变个体时，从当前个体中随机选择一个位置 $i$，并根据 选择不同变异操作的概率 $p_l$ 从突变列表 $l_m$ 中选择一个特定的突变操作 $m$。 然后，在位置 $i$ 上执行 $m$。在突变列表 $l_m$ 中定义的可用变异操作有：

    1. 添加带有随机设置的跳跃层；

    2. 添加具有随机设置的池化层；

    3. 移除选定位置的层；

    4. 在选定位置随机更改构建块的参数值。

Tournament Selection

由于算法执行的效率以及易实现的的特点，锦标赛选择算法是遗传算法中最流⾏的选择策略。在本⼈的实际应⽤中的确此策略⽐基本的轮盘赌效果要好些。他的策略也很直观，就是我们再整个种群中抽取n个个体，让他们进⾏竞争(锦标赛)，抽取其中的最优的个体。参加锦标赛的个体个数成为tournament size。通常当n=2便是最常使⽤的⼤⼩，也称作Binary Tournament Selection.

交叉算子 & 变异算子

  1. 传统的单点交叉仅为长度相等的个体设计。设计的交叉算子用于长度不等的个体。虽然设计的交叉算子很简单，但它可以提高发现CNN结构的性能，实验已证明。

  2. 其次，现有算法使用相同的概率来选择特定的变异操作。在该算法中，所提供的变异操作以不同的概率进行选择。具体而言，为1. “添加跳跃层”突变提供了更高的概率，这将有更高的概率增加CNN的深度。对于其他变异操作，仍然使用相同的概率。这种设计的动机是，更深的CNN将拥有更强大的功能。虽然“添加池层”也能够增加CNN的深度，但通过使用一个池化层，输入数据的维数将减少到一半，从而导致发现的CNN不可用。

我认为总的来说，就是：提供不同的概率选择不同的变异操作集成了一个变异算子。

环境选择

环境选择就是在本代种群中选出个体组成下一代种群。

  1. 首先从当前种群中，通过binary tournament selection选2个个体，选出其中更好的一个fitness更好的存入下一代。
  2. 其次，从父代和子代中选出最佳的个体，检查它是否存入下一代 $P_{t + 1}$ ，如果没存就用最佳的个体取代下一代其中的最差的那个。

原则上，只为下一代选择最佳个体很容易导致算法陷入局部最优。如果不明确地为下一代选择最佳个体，算法将不会收敛。原则上，理想的种群不仅应该包含好的种群，还应该包含相对较差的种群，以增强多样性。为此，二进制锦标赛选择通常用于此目的。然而，仅使用二进制锦标赛选择可能会错过最佳个体，导致算法没有朝着更好的方向进化。因此，我们明确地将最佳个体添加到下一个种群中，这是进化算法中一种特殊的精英策略

Experiments

由于第一类同行竞争对手中最先进的CNN是手工设计的，因此我们主要比较分类精度和参数数量。对于其他两类中的算法，除了分类精度和参数数量之外，我们还比较了用于发现相应CNN的“GPU天数”。特别是，单位GPU日表示算法终止时在一个GPU上执行了一天，这反映了这些算法消耗的计算资源.
