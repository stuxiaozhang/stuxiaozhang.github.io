Evolutionary Architecture Search for Graph Neural Networks

本文最后更新于：2022年9月1日

  1. 在进化搜索框架下提出了一个图神经网络结构搜索问题，旨在优化模型结构和超参数。搜索过程遵循生存拟合原则，该原则具有鲁棒性，可通过迭代进化训练逐步优化GNN模型。
  2. 提出了一种称为遗传GNN的新的进化NAS框架，以从定义良好的搜索空间中自动识别最优GNN架构。与仅优化GNN结构的现有方法不同，执行交替进化过程以同时优化GNN和超参数，直到它们彼此具有良好的拟合。

  • 不变的超参数。除了GNN的结构之外，超参数（例如，学习速率）的轻微变化可能会严重影响具有收敛结构的模型的性能。现有的仅优化具有固定超参数设置的结构变量的方法可能最终得到次优模型。

  • 高计算和低可扩展性。训练递归网络增加了搜索过程的负担。控制器的训练和单个GNN模型都需要大量的运行时计算资源。此外，控制器通常生成候选GNN结构并以顺序方式对其进行评估，这难以扩展到大的搜索空间。

GNN是一系列神经网络模型，可以直接结合图拓扑和节点特征，实现高效的低维节点表示学习。如图3所示，GNN模型的主要思想是每个节点vi通过聚集来自其邻域（例如，本文中的一阶邻域）的特征来生成表示hi。通常，卷积表示学习涉及以下五个GNN结构组件

  1. Attention Function (S1)：当每个节点聚集来自其邻居的特征时，不同的邻居节点可能具有不同的贡献，因为邻居节点和目标节点之间的亲和力不同。注意力函数旨在学习将节点vi链接到节点vj的每个边缘关系ei、j的重要性权重wij。S1表示候选注意力函数的集合。
  2. Attention Head (S2)：如果同时合并多个平行注意力学习，模型可能具有更好的鲁棒性，而不是应用单头注意力。多头注意力允许模型联合关注来自不同节点表示子空间的特征。然后将每个节点vi的多头注意力的多个表示输出串联或平均以生成最终表示hi。S2表示候选关注头编号的集合。
  3. Aggregation Function (S3)：每个节点vi可以具有多个邻域节点，因此需要聚合函数（例如，平均运算）来组合来自多个邻居的特征并形成最终表示hi。S3表示候选聚合函数的集合。
  4. Activation Function (S4)：在导出节点vi的表示hi之后，非线性激活函数（例如，ReLu和Sigmoid）通常应用于平滑hi或变换hi，作为多类节点分类的概率分布向量。S4表示候选激活函数的集合。
  5. Hidden Unit (S5)：隐藏单元控制节点vi的表示hi的维度。对于多层GNN模型，隐藏或中间卷积层中的输出维度可能影响整个GNN模型的表示能力。S5表示候选维度选择的集合。

Genetic-GNN NAS

1. Architecture representation & population initialization

仅优化GNN结构的现有工作可能以次优搜索模型结束，因为超参数的改变可能会严重降低微调的GNN结构。因此，我们主张发展GNN结构和超参数

感兴趣的是优化以下三种类型的超参数，

  1. Dropout Rate (P1)：在训练期间从神经网络中随机丢弃单元（及其连接）。P1表示候选辍学率的集合。
  2. Weight Decay Rate (P∗ 2 )：权重衰减（例如，L2范数正则化）是一种广泛使用的技术，通过限制模型权重的增长来降低复杂性，同时提高神经网络模型的泛化能力。P∗ 2表示候选权重衰减率的集合。
  3. Learning Rate (P∗ 3 )：在基于梯度下降算法训练神经模型时，学习速率决定每次损失变化的速度。较大的学习速率可能导致模型太快收敛到次优解，而较小的学习速率则可能导致优化收敛太慢。P∗ 3表示候选学习率的集合。

超参数P∗ 2和P∗ 3设置一次并在GNN模型的多个层中共享。
