【DeepPath】A Reinforcement Learning Method for Knowledge Graph Reasoning

  • 关键词：基于路径的 KGR；Reinforcement Learning

  • EMNLP 2017.

Basic Idea

1. 解决了什么问题？（what？）

在知识图谱多跳推理任务中:

  • PRA(Path-Ranking Algorithm) 使用基于 restarts 的随机游走推理机制来执行多个有界DFS过程来查找关系路径。结合基于弹性网络的学习，PRA然后使用监督学习选择更合理的路径。
  • 问题：PRA 在一个完全离散的空间中运行，这使得评估和比较KG中的类似实体和关系变得困难。
    • 随机游走推理的一个潜在瓶颈是存在出(入)度比较大的节点，这会显著降低推理速度并影响推理的准确性。

2. 用什么方法解决？（how？）

  • 是第一个用 RL 方法学习 KG 中的关系路径

    • 使用复杂的奖励函数，同时考虑准确性、效率和路径多样性，在寻路过程中提供更好的控制和更大的灵活性

DeepPath

关系推理的具体任务是在实体对之间找到可靠的预测路径。可以将寻路问题描述为一个序列决策问题，可以用 RL agent 解决。

RL for Relation Reasoning

RL 系统由两部分组成（见图1）。第一部分是外部环境 $E$，被建模为马尔可夫决策过程(MDP)，用元组 $<\mathcal{S}, \mathcal{A}, \mathcal{P}, \mathcal{R}>$ 表示:

  • States: $\mathcal{S}$ 是连续的 state space. 每个 state 都会获得 agent 在KG中的位置。选择一个 action 后，代理将从一个实体移动到另一个实体。步骤 t 的状态向量如下所示：

    • $\mathbf{e}t$ 表示当前实体节点的嵌入，$\mathbf{e}{target} - \mathbf{e}_t$ 表示尾实体节点和当前节点之间的距离

  • Actions: $\mathcal{A}={a_1,a_2…a_n}$：所有可能的 actions 集合

  • $\mathcal{P}(S_{t+1} = s’|S_t=s,A_t=a)$：==转移概率矩阵？==

  • $\mathcal{R}(s,a)$：$(s,a)$ 的奖励函数

    • Global accuracy（全局reward）: 对于整体而言，agent 在一系列 actions 后到达目标，就奖励 1.

    • Path efficiency（长度reward）: 观察到短路径比长路径更可靠

      其中，$p$ 是 $r{1} \rightarrow r{2} \rightarrow \ldots \rightarrow r_{n}$

    • Path diversity（路径多样性reward）: 为了鼓励代理找到不同的路径，用当前路径和现有路径之间的余弦相似性定义了多样性奖励函数：

      其中，$\mathbf{p}=\sum{i=1}^{n}\mathbf{r_i}$ 是 $r{1} \rightarrow r{2} \rightarrow \ldots \rightarrow r{n}$ 路径嵌入的和，是新找到的路径实例，$\mathbf{p}_{i}$ 是已有的路径实例。

另一部分是：RL agent，被表示为 policy network $\pi_{\theta}(s, a)=p(a \mid s ; \theta)$，该函数将状态向量 s 映射为所有可能动作的概率分布。神经网络由两个隐藏层组成，每个隐藏层后面是一个 ReLU。使用 softmax 函数对输出层进行规一化。使用随机梯度下降法更新神经网络参数 $\theta$。

Training Pipeline

通常的深度学习模型都只有训练、测试、使用三个步骤，而DeepPath却有预训练、再训练、测试、评价四个部分。

  1. 有监督的预训练：对应 sl_policy.py 中的 train()。利用深度优先搜索在图谱中找到实体对之间的路径，然后用这些路径预训练agent。

  2. 再训练：这一部分主要是利用强化学习的方法训练agent，其中有4种情况会更新agent的决策网络参数。预训练和再训练的网络结构完全一样，但loss不同。预训练的loss与reward无关，而再训练的loss与reward有关，着也造成update更新的2、3两种情况的不同：

    1. 当路径搜索过程中出现无效action的时候。即存在负例 $r_{\mathrm{GLOBAL}} = -1$ 时，针对这些负例更新网络参数。
    2. 路径到达目标实体，即 $done = 1$ 时，利用 $r{\mathrm{GLOBAL}} = 0$ 的正例更新网络参数。 其最终的奖励为 $r{\mathrm{total}} = 0.1r_{\mathrm{GLOBAL}} + 0.9 r_{\mathrm{length}}$
    3. 路径没有到达目标实体，即 $done = 0$ 时，同样用正例来更新网络参数。2. 3. 的差别体现在reward上，3的 $r{\mathrm{total}} =r{\mathrm{GLOBAL}} = -0.05$
    4. 路径没有到达目标实体，用深度优先搜索找到实体对之间的路径，取 $r_{\mathrm{total}}=1$ 进行训练 我没看懂这块…在知乎提问了，有时间再看一下源码吧

  3. 测试：利用再训练得到的agent，找到对应关系的特征路径。虽说是测试，但依然有一次参数更新，利用的就是之前提到的 $r_{\mathrm{diversity}}$。如果agent到达了目标实体，即成功发现了一条特征路径，则利用多样性奖励更新一次参数，让agent下次找到同样路径的概率变小。这里更新的模型是不用保存的，每次寻找特征路径时都会根据当时已有路径的情况来调整。

  4. 评价：虽说是评价，其中依然包含训练部分，即用一部分已知关系的实体对的路径特征向量去训练二分类器，再用测试集的路径特征向量来评价二分类器。这里的路径特征向量很好得，只需要在图谱中判断实体对之间有没有该关系特征路径集合中的路径即可。

Supervised Policy Learning

在实践中，KG 推理的一大挑战是 relation set 可能相当大。如果直接通过试错训练 RL 模型（这是 RL 算法的典型情况），则面临难以收敛的问题，受 AlphaGo 中使用专家指导预训练的启发，本文使用监督的策略学习进行预训练。

对于每个关系，用所有正样本（实体对）的子集来进行监督策略学习。对于每个正样本 $(e{source}、e{target})$，使用双向的 BFS 算法寻找正确连接两个实体的路径。对于每条路径，使用 REINFORCE 更新参数 $θ$：

然而，BFS 是一种偏向于短路径的搜索算法。当插入这些有偏向的路径时，代理很难找到可能有用的更长路径。作者采用了一个简单的技巧，随机选择一个中间节点 $e{inter}$，然后在 $(e{source}, e{inter})$ 和 $(e{inter},e_{target})$ 之间执行两个BFS搜索路径，将搜索的路径合并在一起来训练智能体，监督的策略学习从失败的经验中学习到很多，然后在训练智能体找到期望的路径。

Retraining with Rewards

为了找到由奖励函数控制的推理路径，使用奖励函数对监督策略网络进行再训练。对于每个关系，一个实体对的推理被视为一个事件。

在指定次数内，从源节点 $e_{source}$ 开始

  • agent 根据随机策略 $π(a |s)$ 选择一个关系，这是所有关系的概率分布，以扩展其推理路径。这种关系可能会导致一个新的实体，也可能不会导致任何结果。
  • 无效 actions 导致代理收到 负面奖励。之后，代理将保持相同的状态。由于代理遵循随机策略，所以不会因为重复错误的步骤而陷入困境。
  • 如果成功 或者 未能在在指定次数内到达目标实体，则事件结束。

更新参数：无效 actions 的 Reward = -1。

Bi-directional Path-constrained Search

在KG中，一个实体可以通过一个关系连接到巨量实体，比如 $personNationality^{-1}$: 中国是很多人的国籍，如果规则中包含这样的关系，那路径的中间实体就会超级多，但如果换个方向就没有那么多了，所以就要 bi-directional。
