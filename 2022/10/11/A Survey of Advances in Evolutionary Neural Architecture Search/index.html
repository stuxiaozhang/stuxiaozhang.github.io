A Survey of Advances in Evolutionary Neural Architecture Search

神经结构搜索（NAS）用来自动设计神经网络。NAS可以被视为一个优化问题，在这个问题上，找出最佳的模型体系结构，以在特定任务上实现最佳性能。由于通常很难用连续参数表示模型结构，并且模型结构与其性能之间没有明确的公式，因此该优化问题通常具有非凸和黑盒特性。

Neural Architectural Search

Problem statement

两个方面决定了神经网络的性能，即模型结构 \(M_a\) 和模型参数 \(M_p\)。前者是指网络中的基本单元，包括单元类型（如卷积层、池化层和全连接层）、单元数量以及这些单元之间的连接，定义了模型的能力。后者是关于这些单元中的可训练参数，即连接权重和biases。因此，要为特定任务设计最佳模型，NAS需要搜索与模型参数相关的最佳模型体系结构。它可以表示为方程中的优化问题：

这是一个双层优化问题。上层优化问题意味着在Pareto Front上找到最优模型结构，以最小化验证数据集上的（可能）多个目标函数。下层优化问题是指优化模型参数以最小化训练数据集上的损失函数。还有各种各样的限制：

  1. 一是限制候选解决方案的搜索空间，包括预定义可能单元的类型，以及确定网络中单元的最大数量
  2. 另一个问题是防止发现无效的解决方案，包括无效的架构（例如完全连接的层后面是卷积层）和不满足特定要求的架构，例如模型大小

NAS是一个极具挑战性的优化问题。具体而言，该问题的性质总结如下：

  • 巨大的搜索空间，其中候选模型体系结构随着可能的连接、单元数量和单元类型呈指数增长；
  • 一个黑盒优化问题，因为模型体系结构的性能无法公式化；
  • 一个非凸和不可微的搜索环境，因为模型架构很难用连续参数表示
  • 可能是一个具有各种约束的多目标优化问题。

Existing methods

问题 (1) 经常迭代求解。以这种方式，设计了一个优化器，用于从搜索空间对候选模型架构进行采样; 然后，通过基于梯度的方法在训练数据集上训练这些候选模型，并在验证数据集上进行评估；每个模型的评估都作为反馈，以帮助优化器重现具有更高性能的模型体系结构。当前工作中应用的主流优化方法有三种，即基于RL的方法、基于EA的方法和基于梯度的方法。

Reinforcement learning based methods.

基于强化学习的方法。对于基于RL的方法，设计了基于马尔可夫决策过程的策略，以产生有前途的模型体系结构 [8]，[9]，[23]。具体地，应用策略逐步构建DNN，然后将构建的网络的性能依次作为优化策略的奖励。基于rl的方法的目标是优化策略以产生最高奖励的DNN。例如，在 [23] 中，使用RNN作为策略。RNN接收网络的架构信息，并输出要在其中添加的单元或连接，以逐步生成更大的模型。然后，将生成的模型的性能作为优化RNN的反馈。此过程是迭代的，以使RNN产生更高精度的模型体系结构。

Evolutionary algorithms based methods

将EAs应用于NAS时，将初始化一组模型体系结构。然后，基于每个模型的性能，通过进化操作 (例如重组和突变) 来修改这些模型体系结构，以重现具有更高性能的新个体。最后，选择性能最高的模型作为输出。特别是，在EAs中，为了设计灵活的进化操作，模型体系结构通常由特殊的编码表示

Gradient-based methods

考虑到模型体系结构的搜索空间通常是离散的，因此，如果使用基于梯度的方法，则应将此搜索空间转换为连续的搜索空间。然后，可以计算梯度信息以指导搜索过程。例如，在 [7] 中，通过将所有候选单元的离散架构权重wa ∈ {0,1} 放宽到连续值wa ∈ [0,1]，可以获得梯度信息 ∆ wa来更新wa，并在充分训练后选择具有最高架构权重的单元来构建目标网络。

ENAS

ENAS是应用EAs自动搜索模型架构。EAs包括遗传算法 (GA)，遗传编程 (GP)，进化策略 (ES) 和差异进化 (DE)。所有EAs都遵循一般框架来解决问题，例如初始化，评估，复制和选择。具体来说，一组模型体系结构作为初始种群产生。根据现有的工作，通常的初始策略是随机生成这些模型架构。然后，针对特定任务训练和评估这些模型，并且这些评估(例如，准确性、模型大小和延迟) 是模型的适应度值。之后，将这些进化操作应用于基于种群及其适应度值来再现新的模型体系结构。例如，recombination 重组运算符用于交换两个模型体系结构并产生两个新模型；mutation 突变运算符是关于改变单个模型体系结构 (例如，添加一个跳过连接，删除一个层或调整一个卷积层的大小) 以重现新模型。最后，在选择阶段，从旧种群和复制的模型架构中选择精英，为下一代构建种群。上面提到的过程是迭代单元，满足一些终止标准 (例如，达到最大生成)。

EAs的直接搜索过程使这些算法对于复杂的优化问题很自然。与基于RL的方法相比，EAs搜索的模型体系结构在初始阶段具有更高的精度，这意味着EAs适用于计算受限的情况 。此外，EAs的元参数很少，使得这些算法更简单（Aging Evolution那篇）。与基于梯度的方法相比，尽管EAs在早期很耗时，但通过使用加速技术，例如inheritance 和 one-shot model（Cars那篇），基于EA的方法可以在短时间内找到有前途的模型体系结构。值得注意的是，基于梯度的方法比基于EA的方法对搜索空间的约束更强，这意味着基于EA的方法更适合探索各种搜索空间，包括无限搜索空间。

Advances In Evolutionary Architecture Search

Shallow networks

2002年，NEAT由Stanley等人提出。为了灵活地调整网络体系结构，NEAT使用了图形编码。对于这种编码，有两个部分，即节点基因和连接基因。前者包括神经元的信息，包括它们的激活函数。后者类似于邻接列表，可以表示具有权重的单位之间的连接。图1用于显示NEAT中的网络表示。基于图编码，有效的recombination可以在两个模型之间交换架构，并且可以应用mutation来灵活地插入或消除网络中的神经元或连接。

2007年，HyperNEAT被提出。与原始版本NEAT相比，HyperNEAT还应用了EAs来设计神经网络的体系结构和优化权重，但是它使用indirect encoding来表示模型的权重，而不是real-value encoding，其中可以搜索具有较大尺寸参数的模型。尽管EAs可以灵活地设计模型体系结构，但这些工作主要集中在具有简单神经元的浅前向神经网络上。

Deep networks

EAs在DNN架构优化中的工作可以分为两种类型：

第一种是优化固定连接模式下的架构参数，包括优化每一层的层数和超参数 (例如，卷积层的内核大小和全连接层中的神经元数量)。特定的编码通常表示这些体系结构参数，并应用EAs搜索最佳代码值，以使模型达到更高的性能。体系结构参数的简单表示是数组列表，其中每个元素对应于图层的超参数，列表的长度是层数。通过调整元素的值并插入或消除元素，可以轻松获得具有各种体系结构的DNN。图2用于显示通过EAs优化的CNN架构参数。

第二种是预先定义层及其优化之间的连接。例如，（Structure discovery of deep neural network based on evolutionary algorithms 这篇）应用邻接矩阵来表示全连接层之间的自由连接，并且使用GA来搜索这些基本单元的最佳拓扑；Desell等人 [35] 应用蚁群来设计深RNN细胞中神经元的连接。

然而，目前EAs的应用仍然停留在传统的方法上，即人工预定义网络的部分，然后通过EAs优化剩余的参数，使这些网络达到预测性能。 因此，期望通过创新进一步提高模型体系结构设计的自动化程度。

Innovation in deep networks

Large-scale evolution of image classifiers：为了提高自动搜索的程度，2017年，Google应用EAs对整个CNN进行自动搜索。 本文所用的方法可以看作是一种改进的NEAT方法，它用一个基本单元代替每个节点，只考虑mutation操作来繁殖新的个体。 研究人员只需定义网络中可能的单元，并设计变异操作，就可以全自动地获得DNN。

开创工作吧，将进化与NAS结合。

从模型体系结构表示和进化操作设计两个方面进行了改进。

1、改进模型体系结构表示使模型体系结构的搜索空间更加平滑，可以更有效地进行探索：

  • CGP-CNN：通过将邻接表嵌入到节点基因中，将图编码转换为列表。

  • Hierarchical representations for efﬁcient architecture search：提出了一种分层表示法。 将网络划分为不同级别的子网络，其中高级子网络由低级子网络组成，最低级子网络约为一组基本单元，即卷积层和池化层。 突变可以出现在任何子网络中改变其体系结构。 通过这种表示，可以通过非常简单的进化操作来搜索各种CNNs。 图4用于清晰地显示层次表示。

进化操作设计了更有效的操作来搜索最优模型结构，包括初始化、重组、变异和选择。

  • Evodeep: a new evolutionary approach for automatic deep neural networks parametrisation ：在专业知识的基础上，预先定义了EAs的初始模型结构，这大大减少了搜索空间，可以使EAs更快地收敛到最优模型。

  • EvoCNN：提出了一种特定的分层重组，以在两个不同深度的模型之间交换体系结构。一般的重组操作，只能交换类型相同、位置相同的单位。 改进后的重组操作可以不同类型，不同位置交换，有助于发现不同深度的体系结构。 变异常用于搜索空间中的新区域，但考虑到模型结构的搜索空间很大，这种操作可以搜索不重要的区域。

  • RENAS：设计了一种控制器，将结构改造成一种新的结构或一种性能更高的结构，在探索和开发之间做出了折衷。

  • Aging Evolution：考虑了选择操作中的额外属性年龄，即年龄较大的个体优先从种群中剔除。 这种改进的选择使得种群频繁更新，这可以看作是EAs中的一种正则化，以防止这些算法陷入局部最优。

Booming of deep networks

ENAS的缺点是需要大量的计算资源来搜索整个网络。ENAS的基本搜索范式是一个迭代过程，EAs首先生成新的模型体系结构。 然后对这些模型进行训练和评估，并进一步将这些评估作为反馈来指导EAs生成具有更高性能的模型。 在这个过程中，大量的计算资源被花费在训练这些模型上。 因此，如何用较少的训练对模型进行评估是降低搜索代价的核心步骤。

简单的方法是在训练过程中考虑传统的加速策略，包括减少训练数据量、减少训练时间和预测学习曲线。 对于NAS来说，一个特定的策略是优化代理来预测模型的性能。这个代理可以被看作是从模型的体系结构到其性能的映射。 通过输入与体系结构相关的数据，如层数、层的类型和连接，代理可以在不经过任何训练的情况下输出针对特定任务的模型精度。 因此，可以显著降低NAS的时间成本。 例如，在“Surrogateassisted evolutionary deep learning using an end-to-end random forestbased performance predictor”中，代理是基于随机森林设计的，一组训练数据样本由CNN体系结构（包括DenseNet块、ResNet块和Pooling块的数量）及其精确度组成。 经过代理优化后，EAs生成的模型体系结构可以输入到该代理中，直接输出其精度。 与在搜索空间内训练所有候选体系结构相比，训练代理的时间开销可以忽略不计。

此外，继承是EAS中一种定制的加速策略，通过继承父代的模型参数，无需从头开始训练生成的子代。具体地说，在复制过程（例如，重组和突变）期间，模型参数被保留在相应的基本层中，作为子模型的起点。然后，通过基于梯度的方法的短时间优化运行，这些子代可以达到预期的性能，并在验证数据集上进行评估，以获得他们的适应值。 例如“Simple and efficient architecture search for convolutional neural networks”设计了一种简单的基于遗传变异操作的搜索模型体系结构。以一个经过充分预训练的简单模型为起点，通过添加块或增加通道，获得更深或更宽的子代。 子代被训练，表现较好的子代被选为父母，以繁殖新的个体。 在图5中，有一个关于与继承相关联的重组以再现新架构的例子。 由于不需要从头开始训练，在CIFAR-10数据集上获得约95%的模型体系结构的时间开销减少到几个GPU天（模型大小相差0.5-2个GPU天）。

One-shot 模型是关于一次训练一个超网，并从中推断出最优的模型体系结构。 具体来说，该模型通常分为两个阶段，即training stage 和 deriving stage。在训练阶段构造一个超网，该超网包含搜索空间中的所有候选模型结构，并采用梯度优化方法进行训练。 然后在推导阶段，通过继承模型参数从超网中裁剪出子网，并直接对这些子网进行计算。 最后，选择评价最高的模型架构作为最优解。 在“Single path one-shot neural architecture search with uniform sampling,”中，第二阶段采用EAs作为搜索方法，从超网中截取最优模型结构。

基于cell的体系结构，也用来加快搜索过程。 EAs不是搜索整个网络，而是搜索一个cell（也可以看作是一个子网络），通过对cell进行不同时间的简单堆叠，就可以得到一个完整的网络。 与整个网络相比，cell的搜索空间很小，可以减少时间开销。 例如“Hierarchical representations for efﬁcient architecture search,”中，EAs自动搜索一个cell。 在CIFAR-10上，通过三次堆cell，可以得到一个TOP-1误差为3.6%的CNN；在ImageNet上，通过七次堆cell，可以得到一个较大的CNN,TOP-1误差为20.3%。与使用250个GPU搜索整个网络11天相比，基于cell的架构，在200个GPU上可以将时间成本降低到1.5天。 但与其他提速策略相比，时间成本仍然较高。

Trends

随着先进的EAs技术和加速策略的提出，研究人员致力于以较少的成本实现较高的精度搜索模型体系结构。 例如，在Cars中，One-Shot模型与基于cell的体系结构相结合，其中应用EAs从超网中剪辑单元。 在“Sampled training and node inheritance for fast evolutionary neural architecture search,” 中，inheritance 算子和基于cell的体系结构被一起使用。通过搜索几种单元来构造具有较高精度的模型结构，并通过继承父模型的模型参数，大大减少了时间开销。 通过对这些先进技术的精心融合，EAs目前可以在单个GPU上一天内搜索到基准测试中性能非常高的模型。

另一方面，研究人员正在进一步提高NAS的自动化程度。 虽然可以快速搜索整个网络，但可以找到何种体系结构的搜索空间仍然是由专家设计的。 例如，在Cars中单元由4个节点组成，有8个可能的基本操作。“Single path one-shot neural architecture search with uniform sampling”中，单元有一个20层网络，每层有4个可能的units。 因此，应用EAs自动设计搜索空间。 在“Evolving search space for neural architecture search,”中，原始搜索空间由4个nodes组成，但两个节点之间可能的操作多达27个。 然后，利用EAs从原来的子搜索空间中裁剪出子搜索空间，并根据子搜索空间中的精英模型结构对可能的操作进行评估。 通过迭代应用EAs生成子搜索空间，并对这些运算进行评估，当这些运算的评估值低于预定阈值时将被删除，从而得到一个复杂的搜索空间。

Experiments

然而，这种比较并不完全公平。 第一个原因是，在不同的paper中，搜索空间可能是不同的，在复杂的空间中，即使是简单的随机搜索方法也能以很高的精度找到模型（Random Search and Reproducibility for Neural Architecture Search）。 第二个原因是可以使用各种数据增强技术。 这些数据预处理技术可以提高模型在特定任务上的精度。 因此，如果将使用这些技术的作品与没有数据增强的作品进行比较，这是不公平的。 因此，近年来提出了针对NAS的具体基准，如NAS-Bench-101[56]和NAS-Bench201[57]，将搜索方法应用于相同的搜索空间，以进行公平的比较。
