SQL的数据操作语言DML

发表于 2021-02-07 更新于 2021-02-20 分类于 编程语言 ， SQL

这篇文章整理了有关数据库的 Data Manipulation Language 数据操作语言的相关操作，包括：INSERT，UPDATE 和 DELETE 等等。

DML：数据查询语言

INSERT 语句

INSERT 语句可以向数据表写入记录，可以是一条记录，也可以是多条记录。

  • IGNORE 关键字

IGNORE关键字会让INSERT只插入数据库不存在的记录

  • INSERT 语句中可以包含子查询语句，代表把子查询的内容写入到数 据表

    栗：向技术部添加一条员工记录

UPDATE 语句

UPDATE 语句用于修改表的记录。

练习：

  1. 把每个员工的编号和上司的编号+1，用 ORDER BY 子句完成降序

  2. 把月收入前三名的员工底薪减100元，用LIMIT子句完成

UPDATE 语句的表连接

表连接的UPDATE语句可以修改多张表的记录

1.第一种表连接方式

练习：把底薪低于公司平均底薪的员工，底薪增加150元

2.第二种表连接方式：内连接

练习：把 ALLEN 调往 RESEARCH 部门，职务调整为 ANALYST

3.第三种表连接方式：UPDATE 语句的表连接既可以是内连接，又可以是外连接

练习：把没有部门的员工，或者SALES部门低于2000元底薪的员工，都调往20部门(陈浩没有部门，所以要用左外连接)

DELETE 语句

DELETE语句用于删除记录，语法如下：

练习：

  1. 删除10部门中，工龄超过20年的员工记录

  2. 删除20部门中工资最高的员工记录

DELETE 语句的表连接

1.内连接

练习：

  1. 删除SALES部门和该部门的全部员工记录

  2. 删除每个低于部门平均底薪的员工记录

  3. 删除员工KING和他的直接下属的员工记录，用表连接实现。

    不能直接删掉 t 因为这个表是临时表。可以换成其他写法：ON e.mgr=t.empno OR e.empno=t.empno;

2.外连接

练习：

  1. 删除SALES部门的员工，以及没有部门的员工。

TRUNCATE 快速删除数据表全部记录

DELETE语句是在事务机制下删除记录，删除记录之前，先把将要删除的记录保存到日志文件里，然后再删除记录。

TRUNCATE语句在事务机制之外删除记录，速度远超过DELETE语句
