Python正则表达式

本文最后更新于：2021年2月2日

这篇文章整理了有关 Python 正则表达式的相关操作。

正则表达式

正则表达式：是一个特殊的字符序列，便于检查一个字符串是否与某种模式匹配。

匹配字符串的需要条件：

  • 正则表达式模块：re
  • 匹配字符串的规则
  • 字符串

匹配字符

在此列出一些正则表达式中的常见的特殊字符

特殊字符描述
.通配符。匹配任意字符（不包括换行符）
^匹配开始位置，多行模式下匹配每一行的开始
\A匹配字符串开始位置，忽略多行模式
\Z匹配字符串结束位置，忽略多行模式
\d匹配一个数字， 相当于 [0-9]
\D匹配非数字,相当于 [^0-9]
\w匹配数字、字母、下划线中任意一个字符， 相当于 [a-zA-Z0-9_]
\W匹配非数字、字母、下划线中的任意字符，相当于 [^a-zA-Z0-9_]
\s匹配任意空白字符， 相当于 [
\S匹配非空白字符，相当于 [^

量词符号与组

正则表达式中的符号

特殊字符描述
re1|re2匹配正则表达式 re1 或 re2
^匹配字符串起始部分
$匹配字符串终止部分
*匹配 0 次 或者多次前面出现的正则表达式
+匹配 1 次 或者多次前面出现的正则表达式
{n}匹配 n 次 前面出现的正则表达式
{m,n}匹配 m—n 次 前面出现的正则表达式
[...]匹配 来自字符集的任意单一字符，{n,m}中间逗号不可以有空格
[..x-y..]匹配 x-y范围中的任意单一字符
[^...]不匹配此字符集中出现的任何一个字符，包括某一范围的字符（如果在此字符集中出现）
\将特殊字符无效化

贪婪与非贪婪

在整个表达式匹配成功的前提下，贪婪模式是尽可能多的匹配，非贪婪模式是以最少的匹配字符，默认情况是贪婪模式

  • 0 次 或者多次属于贪婪模式
  • 通过 ？组合变成非贪婪模式 （例如：\d+？）

re 模块的常用函数

findall()

findall(pattern, string, [, flags]) ：查找字符串中所有（非重复）出现的正则表达式模式，并返回一个匹配列表

  • pattern ：匹配规则
  • string ：要匹配的字符串

search()

search(pattern, string, flags=0) ：使用可选标记搜索字符串中第一次出现的正则表达式。成功，返回匹配对象，可以用 groups() 获取数据；失败，返回 None

search( ) 在 string 中进行搜索，成功返回 Match object , 失败返回 None

  • pattern ：匹配规则
  • string ：要匹配的字符串

group() 与 groups()

group(num) ：返回整个匹配对象，或者编号为num的特定子组

groups(num) ：返回一个包含所有匹配子组的元组（匹配失败则返回空元组）

split()

split(pattern, string, max=0) ：根据正则表达式的模式格式符，split 函数将字符串分割为列表，然后返回成功匹配的列表，分割最多操作 max 次（默认分割所有匹配成功的位置）

  • max=0 ：默认为0，匹配到所有能匹配的信息

match()

match(pattern, string, flags=0) ：尝试使用带有可选的标记的正则表达式的模式从头匹配字符串。成功，返回匹配对象；失败，就返回 None。

match() 用来匹配 string 开头，成功返回 Match object , 失败返回 None
