TransH：Knowledge Graph Embedding by Translating on Hyperplanes

本文提出了 TransH，这种超平面的想法解决了同一实体的多关系问题。并且利用一个关系的一对多/多对一映射属性，提出了一个简单的技巧来降低 False Negative 标签的可能性。实验表明，与 TransE 相比，TransH 在预测精度上有显著的改进，并具有与之相当的扩展能力。

Basic Idea

TransE 的问题在于在处理一些关系的映射属性比如自反性、一对多、多对一和多对多这些属性嵌入方面做得并不好。

假设 \((h，r，t)∈∆\) 时 \(h+ r − t = 0\) 的无误差嵌入的理想情况，可以直接从 TransE 模型获得以下结果：

  • 如果 \((h，r，t) \in \Delta\) 且 \((t，r，h) \in \Delta\)，即 \(r\) 是自反关系图，则 \(r = 0\) 且 \(h = t\)。

  • 如果 \(\forall i \in\{0, \ldots, \mathrm{m}\}\)，\((h_i，r，t) \in \Delta\), 即 \(r\) 是多对一关系图, 则 \(h_0=...=h_m\)。

    同样地, 如果 \(\forall i (h，r，t_i) \in \Delta\)，即 \(r\) 是一对多关系图，则 \(t_0=...=t_m\)

问题在于，在 TransE 中，当涉及任何关系时，实体的表示形式相同。而当涉及不同的关系时，则忽略实体的分布式表示形式。

既然实际是表达同一关系不同实体最后通过 TransE 后会趋于一致，那么我直接通过一个中介来进行映射，将同一表示映射成不同向量表示，那么这些向量表示就可以代表不同的实体，就达到了不同实体拥有不同表示的目的。因为关系是不变的，所以想到了将关系作为映射平面，让实体向量向其中映射。

所以作者提出了一种称为超平面上的平移（translation on hyperplanes，TransH）的方法，该方法将关系解释为超平面上的翻译操作。在 TransH 中，每个关系由超平面上的范数向量（\(\mathbf{w}_{r}\)）和超平面上的平移向量（\(\mathbf{d}_{r}\)）表示。对于一个正确三元组 \((h, r, t)\)，它在客观现实中是正确的，期望 \(\mathbf{h}\) 和 \(\mathbf{t}\) 在超平面上的投影通过平移向量 \(\mathbf{d}_{r}\) 以很小的误差连接起来。这种简单的方法克服了 TransE 在处理自反/一对多/多对一/多对多关系时的缺陷，同时模型复杂度与 TransE 基本相同。

TransH

TransH 允许实体在涉及不同关系时具有分布式表示。通过引入投影到关系特定超平面的机制，它可以使实体在不同关系/三元组中扮演不同角色。

同一个实体在不同关系中的意义不同，同时不同实体，在同一关系中的意义，也可以相同。

将每个关系定义在一个独特的平面，在该平面内有符合该关系的 TransH 的表示 \((h,r,t)\)，多加入的代表该平面的法向量完成了将不同实体向平面内和 \(h, t\) 转化的任务，使得同一关系的不同实体拥有不同的表示，但是在关系平面内的投影相同；同一实体可以在不同的关系平面内拥有不同的含义（平面内的投影）

对于正确的三元组来说\((h, r, t) \in \Delta\)，所需满足的关系如图所示。那么对于一个实体 \(h’’\) 如果满足\(\left(h'', r, t\right) \in \Delta\)，在 TransE 中是需要 \(h’’=h\)，而在 TransH 中则将约束放宽到 \(h,h’’\) 在超平面上的投影相同就可以了，也可以实现将 \(h,h’’\) 区分开并且具有不同的表示。

对于关系 \(r\)，TransH 将特定关系的平移向量 \(\mathbf{d}_{r}\) 放置在特定关系的超平面中，而不是放置在实体嵌入的相同空间中。具体来说，对于三元组 \((h，r，t)\)，首先将嵌入 \(\mathbf{h}\) 和 \(\mathbf{t}\) 投影到超平面。投影分别表示为 \(\mathbf{h}_{\perp}\) 和 \(\mathbf{t}_{\perp}\)。如果 \((h，r，t)\) 是正确三元组，期望 \(\mathbf{h}_{\perp}\) 和 \(\mathbf{t}_{\perp}\) 可以通过平移向量 \(\mathbf{d}_{r}\) 连接到超平面，并且误差越小越好。

于是定义在 TransE 中的 \(d(\mathbf{h+r},\mathbf{t})\) ，在 TransH 中的得分函数定义为： \[ d(\mathbf{h+r},\mathbf{t}) =f_{r}(\mathbf{h}, \mathbf{t})= \left\|\mathbf{h}_{\perp}+\mathbf{d}_{r}-\mathbf{t}_{\perp}\right\|_{2}^{2} \] 对于超平面，我们用法向量 \(\mathbf{w}_{r}\) 来表示，并限制 \(||\mathbf{w}_{r}||_2=1\) ，所以可以得到 \(\mathbf{h}\) 在 法向量 \(\mathbf{w}_{r}\) 上的投影为： \[ \begin{aligned} \mathbf{h}_{\mathbf{w}_{r}} &=\frac{\mathbf{h} \cdot \mathbf{w}_{r}}{\left\|\mathbf{w}_{r}\right\|^{2}} \cdot \mathbf{w}_{r} \\ &=\left(\mathbf{h} \cdot \mathbf{w}_{r}\right) \cdot \mathbf{w}_{r} \\ &=\mathbf{w}_{r}^{\top} \mathbf{h} \mathbf{w}_{r} \end{aligned} \] 所以可以得到： $$ {}=-{{r}} =-{r}^{} _{r}

得分函数为： \[ \begin{aligned} f_{r}(\mathbf{h}, \mathbf{t}) &=\left\|\mathbf{h}_{\perp}+\mathbf{d}_{r}-\mathbf{t}_{\perp}\right\|_{2}^{2} \\ &=\left\|\left(\mathbf{h}-\mathbf{w}_{r}^{\top} \mathbf{h} \mathbf{w}_{r}\right)+\mathbf{d}_{r}-\left(\mathbf{t}-\mathbf{w}_{r}^{\top} \mathbf{t} \mathbf{w}_{r}\right)\right\|_{2}^{2} \end{aligned} \] 正确三元组的得分预计会更低，而错误三元组的得分将会更高。TransH 模型参数是所有实体的嵌入 \(\left\{\mathbf{e}_{i}\right\}_{i=1}^{|E|}\)，所有关系的超平面和平移向量 \(\left\{\left(\mathbf{w}_{r}, \mathbf{d}_{r}\right)\right\}_{r=1}^{|R|}\)

Loss Function

作者引入 margin-base ranking function 作为损失函数来训练模型： \[ \mathcal{L}=\sum_{(h, r, t) \in \Delta} \sum_{\left(h^{\prime}, r^{\prime}, t^{\prime}\right) \in \Delta_{(h, r, t)}^{\prime}}\left[f_{r}(\mathbf{h}, \mathbf{t})+\gamma-f_{r^{\prime}}\left(\mathbf{h}^{\prime}, \mathbf{t}^{\prime}\right)\right]_{+} \] 其中 \([x]_{+} \triangleq \max (0, x)\)，\(\Delta\) 是正（正确）三元组的集合，\(\Delta_{(h, r, t)}^{\prime}\) 表示通过破坏 \((h，r，t)\) 构造的错误三元组的集合，\(\gamma\) 是分隔正确和错误三元组的 margin。

除此之外，在最小化 loss function 的过程中，模型还需要遵循三个软约束原则： \[ \begin{array}{l} \forall e \in E,\|\mathbf{e}\|_{2} \leq 1, / / \text { scale } (保证所有实体的embedding都归一化)\\ \forall r \in R,\left|\mathbf{w}_{r}^{\top} \mathbf{d}_{r}\right| /\left\|\mathbf{d}_{r}\right\|_{2} \leq \epsilon, \text { //orthogonal } (保证\mathbf{w}_{r} 和 \mathbf{d}_{r} 正交垂直，保证 \mathbf{d}_{r} 在超平面上)\\ \forall r \in R,\left\|\mathbf{w}_{r}\right\|_{2}=1, / / \text { unit normal vector } (保证单位法向量的模长为1) \end{array} \] 为了体现上面三个约束条件，需要对 loss function 进行修改，加上对公式一和公式二的约束： \[ \begin{aligned} \mathcal{L} &=\sum_{(h, r, t) \in \Delta} \sum_{\left(h^{\prime}, r^{\prime}, t^{\prime}\right) \in \Delta_{(h, r, t)}^{\prime}}\left[f_{r}(\mathbf{h}, \mathbf{t})+\gamma-f_{r^{\prime}}\left(\mathbf{h}^{\prime}, \mathbf{t}^{\prime}\right)\right]_{+} \\ &+C\left\{\sum_{e \in E}\left[\|\mathbf{e}\|_{2}^{2}-1\right]_{+}+\sum_{r \in R}\left[\frac{\left(\mathbf{w}_{r}^{\top} \mathbf{d}_{r}\right)^{2}}{\left\|\mathbf{d}_{r}\right\|_{2}^{2}}-\epsilon^{2}\right]_{+}\right\}, \end{aligned} \]

其中 \(C\) 表示软约束的权重，它也是训练过程中的一个超参数。

Reducing False Negative Labels

训练涉及为正确三元组构造负三元组。TransE 传统的 unif 方法只是通过随机破坏正确三元组来获取负三元组，但是这种随机的方式可能会在训练中引入许多 False Negative 标签。

False Negative：被分为了负样本，但是分错了（事实上这个样本是正样本）。就是说随机产生的错误三元组可能刚好是正确三元组。

TransH 采用不同的方法。在破坏三元组时，通过不同的关系的映射属性(一对多、多对一或多对多)来设置相应的概率来替换头或尾实体。如果关系为一对多，则倾向于更多的替换头实体，如果关系为多对一，则倾向于更多的替换尾实体。这样，减少了产生假负标签的情况。这种方法按比例生成负样本的方法称为 “bern 负采样“。

通过这样的方式，例如 one-many 关系，替换头实体显然更不容易得到正样例（因为只有一种头实体是对的，然而替换尾实体因为对于头实体对应该关系的尾实体更多，说不定就有其他不在此many中的尾实体符合这个关系。

具体而言，在关系 \(r\) 的所有三元组中，首先获得以下两个统计信息：

  1. 每个头实体的平均尾实体数，表示为 \(tph\) (tail entities per head entity)；
  2. 每个尾实体的平均头实体数，表示为 \(hpt\) (head entities per tail entity)。

然后定义了一个伯努利分布 \(\frac{tph}{tph+hpt}\)。给定关系 \(r\) 的正确三元组 \((h,r,t)\)，通过替换头部损坏三元组时的概率为 \(\frac{tph}{tph+hpt}\)，通过替换尾部损坏三元组的概率为 \(\frac{hpt}{tph+hpt}\)

以 FB15k 为例，在 1,345 个关系中，对于每个关系 \(r\)，计算 \({tph_r}\) 和 \(hpt_r\)，如果：

  • \(tph_r <1.5\) 并且 \(hpt_r <1.5\)，则认为 \(r\) 为一对一（一对一的比例为24％）
  • \(tph_r≥1.5\) 并且 \(hpt_r≥1.5\)，则认为 \(r\) 为多对多（多对多的比例为24％）
  • \(tph_r≥1.5\) 并且 \(hpt_r <1.5\)，则认为 \(r\) 为一对多（一对多的比例为23％）
  • \(tph_r<1.5\) 并且 \(hpt_r≥1.5\)，则认为 \(r\) 为多对一（多对一的比例为29％）

伯努利分布

假设抛硬币，正面概率为 p，反面概率为 (1-p)

伯努利实验：抛一次硬币

伯努利分布：抛一次硬币，正面概率为 p，反面概率为 (1-p)，这个就是伯努利分布

n 重伯努利实验：将伯努利实验重复 n 次，就是抛 n 次硬币。

二项分布：n重伯努利试验「成功」次数的离散概率分布，这里的「成功」假设是抛出正面。举个例子：抛硬币 n 次(n 重伯努利试验)，成功次数(抛出正面的次数)为 k 次的概率就是「一个」二项分布概率 \(P\{X=k\}=\left(\begin{array}{c} n \\ k \end{array}\right) p^{k}(1-p)^{n-k}\)。

Experiments

作者对三项任务的相关方法进行了实证研究和评估：链接预测、三元组分类和关系事实提取。这三项任务都从不同的角度评估预测 unseen 三元组的准确性。

Link Prediction

Link Prediction 是为了补全三元组 \((h,r,t)\) 中缺失的 \(h\) 或 \(t\)，即 给定 \((h,r)\) 预测 \(t\)，或给定 \((r,t)\) 预测 \(h\)。该任务不是要求得到最佳答案，而是着重于对知识图谱中的一组候选实体集进行排名。

数据集使用与 TransE 中相同的两个数据集：WN18(Wordnet的子集) 和 FB15k。

Evaluation protocol

遵循与 TransE 相同的 Evaluation protocol，对于每个测试三元组 \((h,r,t)\) ，用知识库中的每个实体 e 替换头实体 h 或尾实体 t，并计算错误三元组上的不相似性分数（根据评分函数 \(f_r\)）。将分数按升序排列，然后得到原始正确三元组的排名。对所有测试三元组进行汇总，评估两个指标：Mean Rank 和 Hits@10。每个正确三元组的能量值排序后的序号求平均，得到的值称为 Mean Rank，Mean Rank 越小越好。计算正确三元组的能量排序后的序号小于10的比例，得到的值称为 Hits@10，Hits@10 越大越好。 这称为“Raw”设置。

注意，同 TransE 一样，如果替换后的三元组仍存在于知识图谱中，因为它也是正确的，那么将其排在原始三元组之前并没有错。为了消除这个因素，在获得每个测试三元组的秩之前，删除了存在于训练集、验证集和测试集中的正确三元组。此设置称为“filter”。把上述基本评价过程得到的结果称为 Raw Mean Rank 和 Raw Hits@10，把过滤掉正确后得到的结果称为 Filtered Mean Rank 和 Filtered Hits@10。

Results

  • 在 WN18 中，简单的模型如 TransE，TransH 都优于其他方法，但是 TransE 要比 TransH 更好点。可能是因为 WN18 中关系数量很少，因此可以忽略不同类型的关系。
  • 在 FB15k 中，TransH 的指标始终优于同类方法。

为了证实，与 TransE 相比，TransH 的这些改进是因为放松了几何假设(relaxed geometric assumption)，因此可以更好地处理自反/一对多/多对一/多对多关系。作者深入研究了不同关系映射类别的详细结果，如表 4 所示。

  • TransE 在 FB15k 中第二好。但是，它在一对多和多对一关系上的相对优势不及在一对一关系上的相对优势。
  • TransH 在一对多，多对一和多对多关系上为 TransE 带来了很大的改进。超出预期的是，一对一的性能也得到了显著改善（> 60％）。==这可能是由于 “graph” 属性所致：实体与关系相关联，因此，更好地嵌入某些部分可以总体上带来更好的结果。==

Triplets Classification

三元组分类是确认给定的三元组 \((h,r,t)\) 是否正确，即在三元组上进行二分类。

该工作使用三个数据集。其中两个与 NTN 中的相同：WN11(WordNet的子集)； FB13(Freebase的子集)。由于 WN11 和 FB13 包含的关系数很少，因此也使用包含更多关系的 FB15k 数据集。详细信息见表2。

Evaluation protocol

遵循与 NTN 相同的 Evaluation protocol：对于三元组 \((h,r,t)\)，如果不相似性得分（通过 \(f_r\)）低于特定于关系的阈值 \(\sigma_{r}\)，则预测为正。否则预测为负。根据（最大化）验证集上的分类准确性确定的特定于关系的阈值 \(\sigma_{r}\)。

Results

  • 在 WN11 上，TransH 优于所有其他方法。
  • 在 FB13 上，NTN 模型表现的最好。在较大的 FB15k 上，TransE 和 TransH 比 NTN 好很多。注意，当实体数接近时，FB15k 的关系数比 FB13 的关系数大得多。这说明 FB13 是一个非常密集的子图谱(大量的实体，少量的关系)，其中实体之间存在很强的相关性。在这种情况下，(NTN 模型)通过张量和非线性变换对实体之间复杂的相关性进行建模有助于嵌入。但是，在 FB15k 这个较稀疏子图谱中，似乎简单地假设在超平面上进行 translation 就足够了，而不必使用复杂的 NTN 模型。
  • 关于运行时间，NTN 的耗时比 TransE / TransH 高得多。
  • 在所有三个数据集上，减少 False Negative 标签（“ bern ”的结果）的技巧有助于 TransE 和 TransH。

Relational Fact Extraction from Text

在本实验中，作者比较了 TransH 和 TransE 对改进关系事实抽取的贡献。该实验包括两个主要部分： text side extraction model 和 knowledge graph embedding.

遵循将知识图谱嵌入的分数与文本侧模型的分数相结合的相同规则，可以获得 TransE 和 TransH 的 PR 曲线，如图2(a) 所示。从图中可以看出，TransH 在改进 Sm2r 方面是“先进”模型，其性能始终优于TransE。图2(a) 中的结果取决于将知识图谱嵌入的得分与文本侧模型的得分相结合的特定规则，实际上规则可能不是最佳方法。因此，图2(a) 不能清楚地展示 TransE / TransH 作为关系事实预测的独立模型的独立功能。为了清楚地展示 TransE / TransH 的独立功能，首先使用 text side model Sm2r 将每个实体对分配给具有最高置信度得分的关系，然后保留那些分配关系不是 “NA” 的事实。对于这些可信的候选事实，仅使用 TransE / TransH 的分数进行预测。结果示于图2(b)，在候选子集上，TransE 和 TransH 的性能均优于 Sm2r。当召回率高于0.6时，TransH 的性能要比 TransE 好得多。

Summary

本文两个贡献：

  1. 提出了 TransH 模型，通过超平面解决了同一实体的多关系问题。
  2. 提出了减少 False Negative 标签的方法，分别统计每个头实体对应尾实体的数量（反之亦然），按占比生成负样例。

TransH 存在的问题：如果两个实体本来就距离很近，那么关于一个关系向量得到的尾向量一定距离很近。然而，每个实体可以有许多属性，不同的关系关注实体的不同属性。关系和实体是完全不同的对象，在同一个语义空间中可能不足以表示它们。
