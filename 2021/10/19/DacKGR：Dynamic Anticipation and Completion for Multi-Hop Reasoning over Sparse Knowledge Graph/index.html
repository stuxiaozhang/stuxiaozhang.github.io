DacKGR：Dynamic Anticipation and Completion for Multi-Hop Reasoning over Sparse Knowledge Graph

DacKGR：Dynamic Anticipation and Completion for Multi-Hop Reasoning over Sparse Knowledge Graph

DacKGR 是 THU吕欣 发表在 EMNLP 2020 上的工作，主要思想还是用强化学习做稀疏 KG 的多跳推理，包括动态预测和动态补全两步。

Basic Idea

该文针对稀疏知识图谱做多跳推理。可以做多跳推理的模型主要有两类：KGE 模型和专门的多跳推理模型。但 KGE 做补全是计算出来的结果，不能给出推理路径之类的解释。而专门的多跳推理模型作用于稀疏知识图谱时面临两个问题：稀疏 KG 包含的信息少，路径选择困难；路径的缺乏使得预测目标实体变得困难。

因此，为了解决在稀疏路径的 KG 进行多跳推理比较困难的问题，文章提出了 DacKGR，包括动态预测（Dynamic Anticipation）和动态补全（Dynamic Completion）两部分：

  1. Dynamic Anticipation：应用 KGE 模型预测尾实体信息用于 agent 选择正确的 action
  2. Dynamic Completion：在路径搜索过程中，动态地向 KG 中添加额外的边（关系），缓解 KG 的稀疏性问题。

Model

RL Framework

近年来，KG 的多跳推理被表述为 KG 上的马尔可夫决策过程（MDP）：给定一个三重查询 \((e_s, r_q, ?)\)，agent 需要从头部实体开始，以最大概率连续选择与当前实体对应的边（关系）作为方向，并跳到下一个实体，直到达到最大跳数。根据之前的工作，MDP 由以下部分组成：

States. 第 t 跳的状态可以定义为 \(s_t=(r_q, e_t,h_t)\)，其中 \(h_t\) 是历史路径的表示。使用 LSTM 对历史路径信息 \(h_t\) 进行编码， \(h_t\) 是第 t 跳 LSTM 的输出。

Action. 对于 \(s_t=(r_q, e_t,h_t)\)，如果有 \((e_t, r_n, e_n)\)，则 \((r_n,e_n)\) 是 \(s_t\) 的一个 action. 状态 \(s_t\) 的所有动作组成动作空间 \(\mathcal{A}_{t}=\left\{(r, e) \mid\left(e_{t}, r, e\right) \in \mathcal{T}\right\}\)

Transition. 如果当前状态是 \((r_q, e_t,h_t)\)，agent 选择 \(\left(r_{n}, e_{n}\right) \in \mathcal{A}_{t}\) 作为下一个 action，则当前状态 \(s_t\) 将变为 \(s_{t+1}=(r_q, e_n,h_{t+1})\)

Reward. 如果找到正确实体，奖励 1. 否则，就是通过 KGE model 的 \(f(e_s,r_q,e_T)\) 的得分。

Policy Network

使用 LSTM 存储历史路径信息。 \[ \begin{array}{l} \mathbf{a}_{t}=[\mathbf{r} ; \mathbf{e}] \\ \mathbf{h}_{t}=\operatorname{LSTM}\left(\mathbf{h}_{t-1}, \mathbf{a}_{t-1}\right) \\ \mathbf{s}_{t}=\left[\mathbf{r}_{q} ; \mathbf{e}_{t} ; \mathbf{h}_{t}\right] \\ \pi_{\theta}\left(a_{t} \mid s_{t}\right)=\sigma\left(\mathbf{A}_{t}\left(\mathbf{W}_{1} \operatorname{ReLU} \left(\mathbf{W}_{2} \mathbf{s}_{t}\right)\right)\right) \end{array} \]

Dynamic Anticipation 动态预测

尽管 KGE 模型不可解释，但在大多数KG上，它们可以比多跳推理模型取得更好的结果。这种现象在稀疏 KG 上更为明显（参见表3中的实验结果），KGE 模型更为稳健 robust，因为它们不依赖 KG 的连通性 connectivity。

作者提出了 Dynamic Anticipation，将 KGE 模型的预测信息引入到到多跳推理模型中，以指导模型学习。具体来说，对于三重查询 \((e_s，r_q,?)\)，使用预训练的 KGE 模型来获得所有实体作为尾部实体的概率向量 \(\mathbf{p} \in \mathbb{R}^{|\mathcal{E}|}\)，其中第 \(i\) 维的值表示 \(e_i\) 是正确尾部实体的概率。

然后状态表示改为： \[ \mathbf{s}_{t}=\left[\mathbf{e}_{p} ; \mathbf{r}_{q} ; \mathbf{e}_{t} ; \mathbf{h}_{t}\right] \] 其中 \(\mathbf{e}_{p}\) 是 KGE models 给出的预测信息。作者使用以下三种策略来生成 \(\mathbf{e}_{p}\)：

  1. Sample strategy. 根据概率分布对一个实体进行抽样，并表示其向量 \(\mathbf{e}_{p}\)。
  2. Top-one strategy. 选择 p 中概率最高的实体。
  3. Average strategy. 根据预测信息 p 的概率分布，取所有实体向量的加权平均值。

实验表明，sample 方式最好。

Dynamic Completion 动态补全

在稀疏 KG 中，头部和尾部实体之间通常没有足够的证据路径，因此多跳推理模型的性能将显著下降。所以提出了 Dynamic Completion，在推理过程中动态地增加每个实体的动作空间。具体地说，对于当前状态 \(s_t\)，其附加操作的候选集可以定义为 \(C_{t}=\left\{(r, e) \mid r \in \mathcal{R} \wedge e \in \mathcal{E} \wedge\left(e_{t}, r, e\right) \notin\right.\mathcal{T}\}\)。需要从 \(C_t\) 额外的 actions 中选择一些概率最高的 actions ，其中概率可以定义为

\[ p\left((r, e) \mid s_{t}\right)=p\left(r \mid s_{t}\right) p\left(e \mid r, s_{t}\right) \] 这里由于候选集 \(C_t\) 太大，计算所有动作的概率将非常耗时，因此采用了一种 approximate pruning strategy 近似剪枝策略：先选定给定状态下概率最高的关系 \(p(r|s_t)\)，然后选给定状态和关系下概率最高的实体 \(p(e | r,s_t)\)。

对于当前状态 \(s_t\)，计算所有关系为 \[p(r|s_t)\] 的 attention value: \[ \mathbf{w}=\operatorname{Softmax}\left(\operatorname{MLP}\left(\mathbf{s}_{t}\right) \cdot\left[\mathbf{r}_{1}, \cdots, \mathbf{r}_{|\mathcal{R}|}\right]\right) \] 然后根据 attention 的值选出前 x 个关系，作为新关系集 \(R_{a d d}=\left\{r^{1}, r^{2}, \ldots, r^{x}\right\}\)

因为额外添加的动作数量为： \[ N_{a d d}=\min (\lceil\alpha N\rceil, M) \] 其中，\(\alpha\) 是控制需要添加的 actions 的比例，\(M\) 是附加动作的最大量。 \(N\) 是当前 state 的动作空间大小。

所以关系选择的个数 x 为：（k是个参数？） \[ x=\left\lceil N_{a d d} / k\right\rceil \] 对于每个 \(r^{i} \in \mathcal{R}_{\text {add }}\) 和 当前实体 \(e_t\)，通过预训练的 KGE 模型预测 \((e_t, r_i, ?)\) 的尾部实体的概率分布为 \(p(e| r_i, s_t)\) 。只保留概率最高的属性，这构成了 \((e_t, r_i, ?)\) 的附加动作 。最后，所有附加动作构成了 \(s_t\) 的 additional action space \(\mathcal{A}_{t}^{add}\)。此附加动作空间将添加到原始动作空间中，并构成新的更大动作空间 \[ \mathcal{A}_{t}=\mathcal{A}_{t}+\mathcal{A}_{t}^{a d d} \] 基于先前的 dynamic anticipation ，dynamic completion 可以生成更精确的动作空间，因为状态包含更多的预测信息。

Policy Optimization

模型使用 REINFORCE 算法训练代理、优化策略网络的参数。可以将这种训练方法理解为和梯度下降相反的训练方法。梯度下降是最小化 loss，这里是最大化 reward： \[ J(\theta)=\mathbb{E}_{\left(e_{s}, r, e_{o}\right) \in \mathcal{K} \mathcal{G}} \mathbb{E}_{a_{1}, \ldots, a_{T-1} \in \pi_{\theta}}\left[R\left(s_{T} \mid e_{s}, r\right)\right] \] 策略网络的参数 \(θ\) 优化如下:（没有加符号，所以是梯度上升）

\[ \begin{array}{c} \nabla_{\theta} J(\theta) \approx \nabla_{\theta} \sum_{t} R\left(s_{T} \mid e_{s}, r\right) \log \pi_{\theta}\left(a_{t} \mid s_{t}\right) \\ \theta=\theta+\beta \nabla_{\theta} J(\theta) \end{array} \]

Experiments

Datasets

为了研究作者的方法在具有不同稀疏度的 KG 上的性能，基于 FB15K-237 构建了三个数据集：EFB15K-237-10%、FB15K-237-20% 和 FB15K-237-50%。这三个数据集分别随机保留 10%、20% 和 50% 的 FB15K237 三元组。此外还从 NELL 和 Wikidata 构建了两个数据集NELL23K 和 WD singer。对于 NELL23K，首先从 NELL 中随机抽样一些实体，然后抽样包含这些实体的三元组以形成数据集。对于来自 Wikidata 的 WD singer，首先在 Wikidata 中找到与 singer 相关的所有概念，然后使用与这些概念对应的实体来构建实体列表。然后，适当地展开实体列表，最后使用实体列表中包含实体的三元组来形成最终的数据集。

Link Prediction Results

从表 3 中可以看到：

  1. DacKGR 模型比以前的多跳推理模型性能更好，而且在越稀疏的数据集上，效果提升越明显。

  2. KGE 模型通常比多跳推理模型取得更好的结果 Why???。这种现象在稀疏 KG 上更为明显。但论文中说只把 KGE 模型作为参考，它们属于不同的类型，不具可解释性。

  3. Sample strategy 比 另两种策略好一些，因为后两者会丢失一些信息。top-one strategy 仅保留概率最高的实体; average strategy 使用 entity vectors 的加权平均，这可能导致不同向量的特征被抵消。

Ablation Study

分别去掉 DA 和 DC 模块观察实验效果，两个模块都可以起到一定的效果，DC 的效果比 DA 的效果好一丢丢，文章的解释是 DA 用于 agent 做出正确的选择，基本不改善 KG 的稀疏性。

DA 和 DC 的关系：DA 只是为 agent 提供动作选择的方向，DC 用 DA 的结果做多跳推理、补全图谱。

Dynamic completion 是否有效？

这部分用于探究额外添加的动作是否被 agent 选择。图中的曲线代表各数据集上的动态补全（DC）的正确率，上面一行是关于训练 epoch，下面一行是关于添加动作的数量占比 \(\alpha\)。DC hits ratio 是选择 \(\mathcal{A}_{t}^{\text {add }} \text { for } s_{t}\)。

从这个图可以看到：

  1. 第一行：对于大多数KG，DC hits ratio 在训练开始时相对较高，然后会急剧下降并趋于稳定。作者认为这是合理的，因为附加动作中存在一些噪音。在开始时，agent无法区分噪声部分并选择与原始动作相同的噪声部分。但随着训练的进行，agent 可以识别噪声部分，并逐渐降低附加动作的选择率。

    对于FB15K-237-10%，DC hits ratio 将先减小后增大。这是因为 10% 的子数据集中 删除了许多三元组，这加剧了数据集的不完整性。在这种情况下，附加动作更有效，并增加了正确推理的概率。证明了模型在稀疏 KG 上表现好。

  2. 第二行：是有关 \(\alpha\) （加入附加动作比例的超参数）的影响，对于大多数据集，DC hits ratio 将随着 \(\alpha\) 的增加而逐渐增加。这与预期的一样，因为 \(\alpha\) 越大意味着：附加 actions 越多，选择这些 actions 的概率越大。注意，在 50% 的子数据集上DC hits ratio 几乎不随 \(\alpha\) 变化。这是因为 50% 的子数据集不太稀疏，添加额外动作对其正确率影响不大。

Summary

文章提出了 DacKGR 来对稀疏知识图谱做多跳推理。包括动态预测（Dynamic Anticipation）和动态补全（Dynamic Completion）两部分：

  1. Dynamic Anticipation：应用 KGE 模型预测尾实体信息用于 agent 选择正确的 action
  2. Dynamic Completion：在推理过程中，动态地向 KG 中添加额外的边（关系），缓解 KG 的稀疏性问题。
