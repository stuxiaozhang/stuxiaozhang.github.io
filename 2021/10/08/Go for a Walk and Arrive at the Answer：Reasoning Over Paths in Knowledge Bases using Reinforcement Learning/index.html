Go for a Walk and Arrive at the Answer：Reasoning Over Paths in Knowledge Bases using Reinforcement Learning

本文最后更新于：2021年10月12日

Go for a Walk and Arrive at the Answer: Reasoning Over Paths in Knowledge Bases using Reinforcement Learning

在知识图谱补全任务中，大多数模型都是依赖低维 representation 对三元组进行打分。本文提出了一种基于输入问题的强化学习（RL）方法，该方法可以有效地在图中搜索答案提供路径，从而消除了对预计算路径的任何需要。给定一个海量知识图，我们学习一个策略，该策略在给定查询 (entity1,relation,?)的情况下，从 entity1 开始，通过选择在每一步取一个带标签的关系边，学习走到答案节点，条件是查询关系和整个路径历史。这将查询-应答任务描述为强化学习（RL）问题，其目标是采取最佳决策序列（选择关系边）以最大化预期回报（到达正确的答案节点）。称 RL 代理 MINERVA 为 ”Meandering In Networks of Entities to Reach Verisimilar Answers.”

MINERVA 的特点：

  1. 首先，MINERVA 具有采用可变长度路径的内置灵活性，这对于回答需要复杂推理链的较难问题很重要。
  2. 其次，MINERVA 不需要预先训练，通过强化学习从零开始在知识图上进行训练；无需进行其他监督或微调，这表明 RL 在 NLP 中的先前应用取得了重大进展。
  3. 第三，作者的基于路径的方法在计算上是高效的，因为通过在查询实体周围的一个小邻域中搜索，它避免了像以前的工作一样在KB中对所有实体进行排序。
  4. 最后，模型的代理发现的推理路径自动形成其预测的可解释来源。

Model

Environment

我们的环境是一个有限视界的、确定性的、部分观测的马尔可夫决策过程，它依赖于从知识库导出的知识图谱 G。在这个图上，现将指定一个确定性的部分观测马尔可夫决策过程，它是一个5元组 \((S,O,a,δ,R)\)

States. 当前所在节点 \(e_t\)，初始节点 \(e_{1q}\)，初始查询关系 \(r_q\)，正确答案 \(e_{2q}\) 四元组。所以 State 表示为 \(S = (e_t,e_{1q},r_q,e_{2q})\)

Observations. 未观察到环境的完整状态。即 除了正确答案 \(e_{2q}\) 无法观测之外 都知道。所以函数定义为 \(O(s=(e_t,e_{1q},r_q,e_{2q}))=(e_t,e_{1q},r_q)。\)

Actions. 由当前节点 \(e_t\) 的关联的三元组 \(\mathcal{A}_{S}=\left\{\left(e_t, r, v\right) \in E: S=(e_t,e_{1q},r_q,e_{2q}), r \in \mathcal{R}, v \in V\right\} \cup\{(s, \varnothing, s)\}\)。处于每个状态的代理都可以选择它希望获取的输出边，并且知道边 \(r\) 和目标顶点 \(v\) 的标签。

Transition. States 结合 Actions 来进行状态转移。转移函数定义为 \(\mathcal{S} \times \mathcal{A} \rightarrow \mathcal{S} \text { defined by } \delta(S, A)=\left(v, \mathrm{e}_{1 \mathrm{q} }, \mathrm{r}_{\mathrm{q} }, \mathrm{e}_{2 \mathrm{q} }\right)\)，其中 \(S = (e_t,e_{1q},r_q,e_{2q})\) , \(A=(e_t，r，v)\), \(\delta : S \times A\)。

Rewards. 如果最后的当前位置是正确答案，给 ==+1== 的最终奖励，否则为0。

Policy Network

结合当前历史行为和状态，对当前可执行运动进行概率评估 \[ R\left(S_{T}\right)=\mathbb{I}\left\{\mathrm{e}_{\mathrm{t}}=\mathrm{e}_{2 \mathrm{q} }\right\} \]

策略 \(\pi=\left(\mathbf{d}_{\mathbf{1} }, \mathbf{d}_{\mathbf{2} }, \ldots, \mathbf{d}_{\mathbf{T}-\mathbf{1} }\right),\mathbf{d}_{\mathbf{t} }: H_{t} \rightarrow \mathcal{P}\left(\mathcal{A}_{S_{t} }\right)\)，历史状态 \(H_{t}=\left(H_{t-1}, A_{t-1}, O_{t}\right)\) 是观察和采取行动的顺序。

Agent 基于 LSTM 对历史行为进行编码，得到历史状态的隐藏表示 \(h_t\)： \[ \mathbf{h}_{\mathbf{t}}=\operatorname{LSTM}\left(\mathbf{h}_{\mathbf{t}-\mathbf{1} },\left[\mathbf{a}_{\mathbf{t}-\mathbf{1} } ; \mathbf{o}_{\mathbf{t} }\right]\right) \] 其中，\(\mathbf{a}_{\mathbf{t}-\mathbf{1} }\) 是在 \(t-1\) 时刻的 action/relation， \(\mathbf{o}_\mathbf{t}\) 是在 \(t\) 时刻的 observation/entity ，\([;]\) 将两者拼接起来。栗：\(\mathbf{a}_{\mathbf{t}-\mathbf{1} }=\mathbf{r}_{A_{t-1} }\) 是 agent 在 \(t-1\) 时刻选择的 关系嵌入对应的 edge 的 label，\(\mathbf{o}_{\mathbf{t} }=\mathbf{e}_{\mathrm{e} }\) 是 agent 在 \(t\) 时刻选择的 实体嵌入对应着的 edge 的 label

根据历史状态的隐藏表示 \(h_t\)，当前观测状态即所处节点表示 \(o_t\) 和初始查询中所含关系表示 \(r_q\) 对下一步的行为做决策 \(d_t\)： \[ \begin{array}{l} \mathbf{d}_{\mathbf{t} }=\operatorname{softmax}\left(\mathbf{A}_{\mathbf{t} }\left(\mathbf{W}_{2} \operatorname{ReLU}\left(\mathbf{W}_{\mathbf{1} }\left[\mathbf{h}_{\mathbf{t} } ; \mathbf{o}_{\mathbf{t} } ; \mathbf{r}_{\mathbf{q} }\right]\right)\right)\right) \\ A_{t} \sim \text { Categorical }\left(\mathbf{d}_{\mathbf{t} }\right) \end{array} \]

Experiments

做了几个实验，验证了：

  1. MINERVA 在小 KB 和大 KB 的查询回答方面具有竞争力

  2. MINERVA 优于基于路径的模型，该模型不能有效搜索知识库或训练查询特定模型

    MINERVA 的表现明显优于 Path-Baseline。这表明，基于一组随机采样路径进行预测的模型不如 MINERVA，因为它要么在随机行走过程中丢失重要路径，要么无法聚合所有k条路径的预测特征，其中许多路径与回答给定查询无关。后者类似于远程监督问题，即重要证据在大量无关信息中丢失。然而，通过以查询关系为条件采取每一步，MINERVA 可以有效地减少搜索空间，并将重点放在与回答查询相关的路径上。

  3. MINERVA 不仅可以用于格式良好的查询，还可以轻松处理部分结构化的自然语言查询

  4. MINERVA 具有很强的长链推理能力

  5. MINERVA 具有很强的训练能力，并且推理时间更快。

Summary

本文提出了 MINERVA 模型，使用了 Reinforcement Learning 的方法，根据 query 在图谱上游走获得 path，抵达目标节点后停止，该方法可以通过在知识图谱上游走推理 path 获得答案，不需要对所有实体进行枚举。
