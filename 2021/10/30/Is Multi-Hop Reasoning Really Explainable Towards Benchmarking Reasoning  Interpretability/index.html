Is Multi-Hop Reasoning Really Explainable? Towards Benchmarking Reasoning Interpretability

本文最后更新于：2022年1月5日

Is Multi-Hop Reasoning Really Explainable? Towards Benchmarking Reasoning Interpretability

这篇论文是今年新出的，在 arxiv 上，但还未公开发表，但根据作者吕欣历年的习惯，八成是要投今年的 EMNLP。

果然中了 EMNLP...

文章的主要工作是提出了一个 benchmark：BIMR (Benchmark to detect the Interpretability of Multi-hop Reasoning)，用于对多跳推理的效果进行评估，并用该 benchmark 对 9 个 baseline 模型进行了测试，验证了 A-benchmark 的有效性 和 R-benchmark 的泛化性。

Problem

以前的 KG 的多跳推理，大多数都关心模型性能，而忽略了对路径可解释性的评估。

很多多跳推理模型给出的路径并不合理，如上图下面的例子，尽管模型预测出了正确的尾实体，但是给出的推理路径完全不合理，目前对多跳推理效果评价的方法只有 case study，很少有人做可解释性的评估，因此文章提出了量化的评价指标 BIMR，用于评价多跳路径的可解释性。

2. Evaluation Framework

计算模型的可解释性分数基于三个指标定义：Path Recall (PR)，Local Interpretability (LI)，Global Interpretability (GI).

2.1 Path Recall (PR)

路径召回（PR）表示测试集中，模型可以通过至少一条从头部实体到尾部实体的路径找到三元组的比例。 \[ \mathrm{PR}=\frac{\sum_{(h, r, t) \in \mathcal{T}^{\text {test }}} \operatorname{Cnt}(h, r, t)}{\left|\mathcal{T}^{\text {test }}\right|} \] 其中，\(\operatorname{Cnt}(h, r, t)\) 是一个指示函数，表示模型是否可以从 \(h\) 到 \(t\) 中找到路径。如果至少可以找到一条路径，则函数值为 1，否则为 0。

Path Recall 的意义在于：对于大多数模型，不是每个三元组都能找到从头部实体到尾部实体的路径。没有来自 \(h\) 到 \(t\) 的路径，或者可能无法为每个 rule 匹配 KG 上的真实路径。

2.2 Local Interpretability (LI)

局部可解释性（LI）用于评估模型找到的路径的合理性。 \[ \mathrm{LI}=\frac{\sum_{(h, r, t) \in \mathcal{T}^{\text {test } } } \operatorname{Cnt}(h, r, t) S(p)}{\sum_{(h, r, t) \in \mathcal{T}^{\text {test } } } \operatorname{Cnt}(h, r, t)} \] 其中，\(p\) 是模型找到的从 \(h\) 到 \(t\) 的得分最高的路径，\(S(p)\) 是该路径的可解释性得分

2.3 Global Interpretability (GI)

全局可解释性（GI）评估模型的整体可解释性，因为 LI 只能表达由模型发现的路径的合理程度，但是它不能考虑可以找到多少路径。 \[ \mathrm{GI} = \mathrm{PR} \cdot \mathrm{LI} \] LI 可以反映可以找到的所有路径的可解释性，而 GI 评估模型的整体可解释性。

3. Approximate Interpretability Score

构建 benchmarks 来定量评估解释性。然而，由于路径数量巨大，评估框架中的 \(S(p)\) 很难获得。所以在此之前，先进行：路径收集 和 近似策略。

3.1 Path Collection

Path Collection 目的是收集所有从 \(h\) 到 \(t\) 可能的路径，以便评估框架能够覆盖多跳推理模型的各种输出。

  1. 针对训练集中的每个 \((h,r,t)\)，添加反向三元组 \((t,r^{-1}, h)\)
  2. 对测试集的每个 \((h,r,t)\)，使用 BFS 搜索从 \(h\) 到 \(t\) 长度为 3 的所有路径。

最后生成了包含约1600万条路径的最终路径集，它涵盖了多跳推理模型可能发现的所有路径。

3.2 Approximation Optimization

Approximation Optimization 目的是为了避免在大路径集 P（即1600万）上进行不切实际的注释或计算。作者发现路径的可解释性主要来自规则 rule，而不是特定的实体。所以将每个路径 \(p\) 抽象为其相应的规则 \(f\)，并使用规则的可解释性得分 \(S(f)\) 作为路径可解释性得分 \(S(p)\)， \[ S(p) \approx S(f) \] 对于一条路径 \(p\) \[ (h,r,t)←(h, r_1, e_1) ∧ (e_1, r_2, e_2) ∧ (e_2, r_3, t) \] 根据规则 \(f\) 将 \(p\) 转换为： \[ r(X, Y)←r_1(X, A_1)∧ r_2(A_1, A_2)∧r_3(A_{2}, Y) \] 转换之后，将路径集 \(\mathcal{P}\) 转换为一组规则集 \(\mathcal{F}\)。因为这些规则是独立于实体的，所以规则集 \(\mathcal{F}\) 的大小减少到 96019，只需要给出每个规则 \(f \in \mathcal{F}\) 的可解释性分数 \(S(f)\) 用于构建 benchmark 即可。

4. Benchmark with Manual Annotation (A-benchmark)

作者用可解释性分数手动标记每个规则，以形成手动注释的基准 (A-benchmark)。该基准的具体构建过程可以分为两个步骤，即剪枝优化和手工标注。

4.1 Pruning Optimization

为了在不影响最终结果的情况下节省注释成本，作者提出了一种剪枝策略。Rule mining 方法可以在 KG 上自动挖掘规则，并给每个规则一个置信度。作者的剪枝策略基于这样的假设：那些不在 Rule mining 方法所挖掘的规则列表中的规则，或者那些置信度非常低的规则，可解释性分数更低。

规则怎么挖掘的？没有看过 Rule mining 的论文...

作者使用 AnyBURL 在训练集上挖掘规则，并获得 36065 条 rules 的规则集 \(\mathcal{F}^A\) 。根据规则集 \(\mathcal{F}^A\) 的 rule 及其置信度得分，将整个规则集 \(\mathcal{F}^A\) 分为三组：高置信度规则（H rules）、低置信度规则（L rules）和其他规则（O rules）。每组中随机抽取500条规则，并邀请专家用可解释性分数手动标记它们。规则的分类标准和标签结果如表所示。

三类规则的分类标准和统计信息，其中 \(C(f)\) 是规则 f 的置信度得分，#Rules 表示规则的数量，#Paths 表示与此类型规则对应的路径数量，Avg Score 表示此类规则的平均可解释性得分。0.01是用于选择低置信度规则的阈值，被规则挖掘方法广泛采用。

从表2可以发现 L rules 的平均解释性得分远低于 H rules。虽然 O rules 的平均分数没有 L rules 的平均分数小，但 O rules 对应的路径太少（0.8M），而规则的数量相对较大（75027）。因此，O rules 可以被视为“长尾”规则。

为了节省标注成本，并且尽可能不影响最终的评价结果，只手动标注 H rules 的可解释性得分。其他规则，使用此类规则的平均分数作为其可解释性分数，即认为所有 L rules 的可解释性得分为 0.005，所有的 O rules 可解释性得分为 0.069.

长尾：大概的意思就是虽然尾部很小，但是关注“尾部”产生的总体效益甚至会超过“头部”。在这里应该是，虽然 O rules 的的路径很少，但是它产生的规则数量很大。

4.2 Manual Annotation

因为注释者很难直接注释规则的可解释性分数，因此对于每个 H rules，随机抽取最多10条对应的真实路径进行标记，用 与该规则对应的路径的平均可解释性得分 作为该规则的得分： \[ S(f)=\frac{1}{\left|P_{c}(f)\right|} \sum_{p \in P_{c}(f)} S^{A}(p) \]

其中，\(|P_{c}(f)|\) 是规则 \(f\) 的所有采样路径，\(S^{A}(p)\) 是通过手动注释获得的路径的可解释性得分, 因为无法对所有路径的可解释性得分进行注释，对于等式 8 中注释的少数路径，也就是每个 H rules，随机抽取最多的那10条对应的真实路径，可以直接获得它们的 \(S(p)=S^{A}(p)\)，而对于大多数其他路径，也就是 H rules 下 没被抽取到的真实路径，用等式 5：\(S(p) \approx S(f)\) 获得它们的近似解释性得分。可解释性分数让注释者选择合理、部分合理和不合理 \((1，0.5，0)\)

5. Benchmarks with Mined Rules (R-benchmark)

R-benchmark：使用 Rule mining 方法来挖掘训练集上的 rules，这些挖掘的规则形成一个规则集 \(\mathcal{F}^*\)。可以使用规则的置信度作为可解释性得分。但会有另一个问题：规则的置信度和可解释性得分之间没有校准。

解决方式：通过两个阈值，使用三个分类来标记规则的可解释性得分： \[ \operatorname{Type}(f)=\left\{\begin{array}{l} \text { unreasonable, } c(f)<h_{1} \\ \text { partially reasonable, } h_{1} \leq c(f)<h_{2} \\ \text { reasonable, } c(f) \geq h_{2} \end{array}\right. \] 其中，\(c(f)\) 是规则的置信度，使用 Micro-F1 分数来找到最佳的 h1 和 h2。

Experiments

Link prediction

作者主要做了链接预测和可解释性评估。

  1. 在本文更关注的可解释性方面，AnyBURL 取得了几乎最好的结果，PR 值达到 98，这表明几乎测试集中的所有三元组都可以通过 AnyBURL 找到一条真实路径。

  2. 对于多跳推理模型，在 PR 方面 与 AnyBURL 没有太大的差距。

  3. NeuralLP 在所有模型中获得最高的 LI 分数，并且比其他模型高得多，但这并不意味着 NeuralLP是一个高度可解释的模型，因为它的 PR 分数只有 10.2，因为它使用 高置信度的 rule 作为解释。对于大多数三元组，它不能给出与实际路径对应的规则。

    所以必须得有 GI 来反映整体的可解释性。

  4. 在所有模型中，AnyBURL 的 GI 得分最高，远远高于第二名。RuleGuider是一种基于多跳的改进模型。它在 reward 中加入了规则（用 AnyBURL 来挖掘）的置信度信息，提高了模型的可解释性。从实验结果来看，这种改进确实是有效的

  5. 当前的多跳推理模型确实具有一定的可解释性。然而，与最好的 基于规则的推理模型 AnyBURL 相比，多跳推理模型在可解释性和链路预测性能方面仍然存在一些差距。这表明目前的多跳推理模型存在许多不足，仍然落后于基于规则的最佳推理模型。

Manual Annotation vs. Mined Rules

ABS-DIFF-AVG表示该基准的可解释性得分与 golden score 之间绝对值的平均值。

可以看到，A-benchmark 可以准确地评估模型的可解释性。R-benchmark 无法提供准确的结果。然而，它有自己的一些优势。例如，它可以应用于其他KG，具有更大的通用性。此外，它还可以为不同模型的解释性比较提供参考指标。

Summary

以前的 KG 的多跳推理，大多数都关心模型性能，而忽略了对路径可解释性的评估。本文提出了一个定量评估多跳推理模型可解释性的框架。基于这个框架，对一个数据集进行注释，形成一个名为 BIMR 的 benchmark，该基准可以准确地评估可解释性。实验结果表明，A-benchmark 与 golden truth 的结果相似，可以作为评估模型可解释性的指标。R-benchmark 的准确度不高，但是作者解释说它具有通用性，可以推广到大多数KG。

另外，作者提出了未来的工作可以研究：如何更好地将 挖掘出的规则 融入到 多跳推理模型 中，以获得更好的性能和可解释性。
